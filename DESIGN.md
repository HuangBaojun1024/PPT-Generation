# DESIGN.md — 决策日志

> 输入一个 JSON（topic / brief / audience），输出一套 25–30 页、风格统一、有叙事线的 `.pptx`。
> 考核维度：美观度 / LLM 成本 / 生成速度，交付 `beauty-max`（拉满美观）与 `balanced`（trade-off）两版。
> 完整技术方案见 `docs/pipeline-proposal.md`，本文只讲「决策」。

---

## 1. 架构图 + 数据流

核心判断：**不让 LLM 直接吐 pptx 坐标，而是让它写 HTML/CSS**——LLM 的前端排版能力远强于摆 pptx 坐标，且 HTML 能被浏览器精确测量，天然自带质检能力。pptx 只是最后一道「格式转换」。

```
输入：一个 JSON { 主题, 简介, 受众 } + 两个开关（视觉档位 / 模型组）
        │
 ① 规划（一次 LLM 调用，高推理）      ← 全流程唯一的"贵"调用
    干什么：读懂主题和受众，定全局叙事线、整套配色与字体、选一套版式模板
    产出：一份"全局蓝图"——叙事弧 + 章节结构 + 风格色板 + 选中的模板
        │
 ② 分镜（LLM 调用，按章节并行）
    干什么：把蓝图铺开成 25–30 页，逐页定内容和页型
    产出：每页的文案 + 页型（封面 / 要点 / 时间线…）+ 素材需求（要不要配图、图标）
        │
 ③ 策略（纯代码，不花钱）             ← 确定性规则
    干什么：给每页挑版式、列素材清单，并把有限的生图预算分给最该配图的几页
    产出：每页一份"渲染方案"
        │
    ③.5 素材并行预取：按方案并行生成配图 / 图标 / 图表
        │
 ④ 渲染（LLM 调用，逐页并行）
    干什么：拿每页方案 + 全局风格 + 素材，生成一页 HTML
    产出：25–30 段单页 HTML（全套共用同一份样式表，天生一致）
        │
 ⑤ 评估与修复
    第一层（纯代码）：用浏览器实测每页有没有文字溢出 / 重叠 / 字太小 → 有问题只重做那一页
    第二层（可选，一次 LLM 审查）：把整套缩略图拼成一张大图，挑出"跳戏"的离群页 → 只返工问题页
    产出：修好的整套 HTML
        │
 ⑥ 合并导出（纯代码）
    干什么：把每页 HTML 转成一页 PPT（背景保真 + 文字可编辑，对不齐时整页截图兜底）
    产出：final.pptx + 成本/时延报表 + 质检记录
```

**LLM 只花在 3 个环节**：① 规划 1 次、② 分镜约 5–7 次（并行）、④ 渲染 25–30 次（并行）+ 返工页。③ 策略和 ⑥ 合并是零 LLM 的纯代码。设计原则一句话：**贵的推理只花在 1 次全局规划上，其余都是可并行的便宜调用或零成本代码。**

`--profile`（视觉预算）与 `--preset`（各角色用什么模型）是**两个正交开关**，可自由组合。

## 2. 模型选型

**逻辑：每个环节先锚定该领域最强模型（对齐 [arena.ai 榜单](https://arena.ai/leaderboard)）顶起效果上限，再逐个下探到高性价比模型找平衡点。**

| 角色          | 最强锚点                   | 为什么是它                            | 下探替代                                   |
| ------------- | -------------------------- | ------------------------------------- | ------------------------------------------ |
| image 生图    | gpt-image-2                | 图像生成榜首                          | 无等效更省项                               |
| coder 写 HTML | claude-fable-5             | Web 前端榜首，美观度上限由它定        | gpt-5.5：构造约束下实测也合格              |
| reviewer 审查 | claude-opus-4-7 (thinking) | 图转代码榜首，跨页比对最稳            | 关闭：一致性靠构造兜住，边际收益撑不起价格 |
| planner 规划  | gpt-5.5 (high)             | 中转站性价比最高；只调 1 次值得高推理 | 已是首选                                   |
| writer 分镜   | gpt-5.5 (low)              | 同上；结构有 zod 兜底                 | glm-5.1 降级线                             |

**gpt-5.5 是高性价比主力**（中转站里最划算）。落地就是逐步下探：coder 从 fable-5 换成 gpt-5.5 实测合格，reviewer 直接关掉省一笔。最终 5 套 demo 全用 `test` preset（三角色全 gpt-5.5 + reviewer 关），够用又便宜；full preset（fable-5 + opus-4.7）留给要把美观度再顶一档的场景。

## 3. 风格一致性怎么保的（核心难点）

**核心：把一致性做成生成前的约束，不是生成后的审美判断。** 每页不各自发挥再靠 reviewer 纠偏，而是从同一主题、同一模板、同一份 CSS、同一素材规范里「编译」出来。实测只靠构造约束 + L1 检查就够，L2 reviewer 设计了但没开。

五层收敛，一层压一层：

1. **全局定调**：planner 一次性定叙事/色板/字体，并从 3 套 LayoutMap 选一套模板。后面 25–30 页只能在这套体系里选 pageType/variant，不能每页自创风格。
2. **样式单一源**：DeckPlan 编译成唯一 `tokens.css`，全套共享。颜色/字号/间距/圆角/安全边距/网格全走 CSS 变量——LLM 写页面结构，不重定义视觉系统。
3. **prompt 注入约束**：coder 拿到的是「页面方案 + 全局风格 + LayoutMap 约束 + token 规则」；生图 prompt 拼固定 Style Anchor（风格描述 + 色板 hex + 负面清单 no text/watermark/logo）。输入侧就共享审美语义。
4. **素材统一口径**：图标只用 Phosphor 一库一字重（beauty-max=duotone / balanced=regular），`currentColor` 继承 token；生成图在 HTML 层叠主色蒙层，强行拉回同一色系，消除 fullImage 页与模板页断层。
5. **检查兜底**：L1 用 Playwright DOM 测量查对比度/字号/边距/溢出/装饰压字（机器能判的硬问题）；L2 reviewer（本次关）拼图找配色/密度跳戏的离群页。

一句话：**主题在 planner 收敛，版式在 LayoutMap 收敛，样式在 token 收敛，素材在图标库/蒙层/Style Anchor 收敛，检查只抓漏网。**

## 4. 多样性怎么保的

一致性锁「视觉**系统**」（色板/字体/图标库/网格）；多样性放「页面**变体**」。三层制造差异又不破系统：

- **模板**：3 套 LayoutMap（aurora 咨询风 / matcha 杂志风 / midnight 深色技术风），planner 按 topic+audience 选，不同输入进不同模板。
- **叙事**：章节和页序是 planner 现场设计的，不是固定套路——「年度复盘」长成"时间线 + 得失盘点"，「说服 CEO 用 Rust」长成"痛点 → 对比 → 收益"，「京都两日游」长成"场景 + 清单"。结构不同，页型自然不同。
- **变体**：同一 pageType 多个 variant，配色主色域内派生，封面/章节生图各不相同。

产出是「同系不同款」，不是「同模板填空」——这也是 5 套 demo 彼此差异明显的原因。

## 7. AI 协作复盘

**关键方案我定，AI 填实现**

- **HTML+CSS 作中间产物** + **LayoutMap 模板**（Map 描述页型→区块约束）——决定美观度上限和一致性根基，我提的。
- **六层主流程**（规划→分镜→策略→渲染→评估→合并）+ **多级检查 + 分类返工**（样式问题回渲染层、素材问题回策略层，单页级不重跑全流程）——我给的思路。
- 一致性的「**全套统一样式**」（落地为 tokens.css）和「**生图注入统一风格 prompt**」（落地为 Style Anchor）——我想的。

**AI 提的、我确认**

- **Playwright** 一个引擎包渲染/测量/截图/质检的闭环——AI 在「HTML 中转」上补的关键实现，也是 L1 零成本的前提。
- **行级文字覆盖**：Range API 逐字符取字形矩形、按行分组、每行一个文本框——比我原想的「整段一框」对齐精度高得多。
- 一致性补充手段：图片主色蒙层、Phosphor 单库单字重、「构造优先、评估兜底」的分层。

**AI 提的、我推翻**

- AI 想 reviewer 常开、逐页精修。我改成可开关 + 拼图审（逐页 = 30 倍成本），本次直接关，用 L1 + 构造扛一致性——够用。
- AI 想 planner 也用低推理省钱。我否：planner 只调 1 次、是叙事总闸门，值得高推理；要省该省生图张数。

**我对 AI 输出的核验/兜底**

- **HTML转PPT表格文字不居中**：细节丢失
- **HTML中部分装饰错乱**：细节丢失
